# Phase 1: Linux Administration & Essentials 🐧

Welcome to the **Linux Phase** of the DevOps Bootcamp! This module is designed to take you from a beginner to a confident Linux user, focusing exactly on the tools, commands, and concepts that a DevOps Engineer uses every single day in the real world cloud environment.

## 📖 Module Overview 

Throughout this phase, we cover 7 crucial topics. You can explore the highly-detailed, easy-to-understand notes for each topic in their respective files below:

### [1. Introduction to Linux](1-linux-essentials.md)
* **Topics Covered:** What is Linux, the Kernel, the GNU project, and why Linux is the absolute industry standard for servers and Cloud computing. We also break down the "layered cake" architecture of the Operating System.

### [2. The Linux File System Hierarchy](2-linux-essentials.md)
* **Topics Covered:** A visual mapping of the Linux file system (The "Office Building" analogy). We explore the meaning of the Root `/` directory, and where critical files live (like configurations in `/etc` and logs in `/var`).

### [3. Working with Files & Text Editors](3-linux-essentials.md)
* **Topics Covered:** Creating files instantly (`touch`), reading them (`cat`), and routing text into files via Redirection (`>`). We also dive into terminal text editors (`nano` and `vi` modes!), plus how to safely view GIANT server logs without crashing your computer (`less`, `head`, `tail`).

### [4. Navigation & Directory Operations](4-linux-essentials.md)
* **Topics Covered:** How to "drive" around the terminal (`cd`, `pwd`), list files including hidden ones (`ls -la`), and safely move/delete items (`mv`, `cp`, `rm -r`). It also features DevOps survival skills for checking a full hard drive (`df -h`, `du -sh`).

### [5. Systems, Services & Software Management](5-linux-essentials.md)
* **Topics Covered:** The "Great Divide" in Linux (`apt` for Ubuntu vs `yum` for RedHat), verifying the server's identity (`hostname`, `cat /etc/os-release`), and the modern way to start/stop background applications using `systemctl`.

### [6. Users, Links & Archives](6-linux-essentials.md)
* **Topics Covered:** How to create users and assign them to groups securely. We break down the difference between a Soft Link (Shortcut) and a Hard Link (Clone). Finally, we pack up and compress massive files using `tar` and `gzip`.

### [7. File Permissions & Ownership](7-linux-esssentials.md)
* **Topics Covered:** The most critical building block of Linux security! We visually decode `ls -l` outputs (`drwxr-xr-x`), translate `chmod` permissions into simple math (the meaning of `chmod 755` vs `chmod 777`), and review file ownership (`chown`, `chgrp`).

---

> **💡 The DevOps Mindset:** As you go through these notes, remember that you do not need to memorize every single command option perfectly. Your goal is to understand *what is possible*, how to navigate the system safely, and where to look when a server breaks! 🚀
